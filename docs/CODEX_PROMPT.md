# Codex 协作提示词约定

后续在本项目中使用 Codex 协作时，默认遵守以下约定：

- 代码注释使用中文。
- 文档说明使用中文。
- 面向开发者的提示词、任务拆解和实现说明使用中文。
- 默认把项目维护者当作 Python 初学者来解释，给类、函数、关键属性、复杂分支补充清晰中文注释。
- 注释要解释“为什么这样写”和“这个对象在流程里负责什么”。
- 新增 Pydantic 模型时，优先给类写 docstring，并给关键字段写 Field 中文说明。
- 新增函数或方法时，优先写中文 docstring，说明参数、返回值和调用时机。
- 保留必要的英文技术名词，例如 Runtime、Adapter、Workspace、MCP Gateway、Docker、Compose。
- 对外部协议字段、包名、类名、函数名等代码标识符不强行翻译。

建议提示词：

```text
你是本项目的 Codex 编程协作助手。后续所有代码注释、开发说明、任务拆解和项目内提示词默认使用中文；默认把项目维护者当作 Python 初学者来解释，给类、函数、关键属性、复杂分支补充清晰中文注释，重点解释“为什么这样写”和“这个对象在流程里负责什么”。新增 Pydantic 模型时给类写 docstring，并给关键字段写中文 description；新增函数或方法时写中文 docstring，说明参数、返回值和调用时机。保留必要英文技术名词和代码标识符。实现前先阅读现有代码与文档，优先遵循 README.md、TODO.md 和 docs/CODEX_PROMPT.md 中的架构约定。
```
